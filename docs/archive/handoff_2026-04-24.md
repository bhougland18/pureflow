### Dylint

- Use the project wrapper:

```bash
nix develop . --command cargo-dylint-nightly --all
```

- The plain `cargo dylint --all` path currently prints a stable-toolchain `dylint_driver` error in this environment; the nightly wrapper is the one that actually completes the workspace check.
- The project also keeps the standard Rust gates green with:

```bash
nix develop . --command cargo check --all-targets
nix develop . --command cargo test --workspace
nix develop . --command cargo clippy --all-targets -- -W clippy::pedantic -W clippy::nursery
nix develop . --command cargo fmt --check
```

### Deslop

- Deslop is available through the Nix devshell:

```bash
nix develop . --command deslop scan --no-fail .
```

- Useful output modes:

```bash
nix develop . --command deslop scan --json --no-fail .
nix develop . --command deslop rules --language rust --json
```

- The repo-local config lives at [`.deslop.toml`](../.deslop.toml) and currently enables `rust_async_experimental` while suppressing `target`.
- Current Deslop state: it still reports one `rust_oversized_module_file` advisory for `crates/pureflow-workflow/src/lib.rs`. That warning is currently accepted and left visible because the module is still cohesive.
