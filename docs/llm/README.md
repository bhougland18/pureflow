# Pureflow LLM Retrieval Notes

This directory contains compact, topic-oriented documentation intended for retrieval use.

## Authoring Rules

- Prefer stable facts over implementation trivia.
- Keep sections short and heading-driven.
- Use explicit terminology and avoid narrative filler.
- Split by topic; one primary concern per file.

## Initial Topics

- `architecture-overview.md`
- `runtime-lifecycle.md`
- `contracts-and-boundaries.md`
- `tooling-and-doc-generation.md`

## Reading Order

If you need a quick architecture answer, start with `architecture-overview.md`.
If you need execution behavior, go to `runtime-lifecycle.md`.
If you need boundary or compatibility facts, use `contracts-and-boundaries.md`.
If you need doc generation or diagram workflow details, use `tooling-and-doc-generation.md`.
