## Recent Work

`pureflow-workflow-format` now owns the external workflow format boundary:

- raw workflow structs live in [crates/pureflow-workflow-format/src/lib.rs](/home/ben/code/pureflow/crates/pureflow-workflow-format/src/lib.rs)
- `pureflow_version` is required and currently supports only `"1"`
- missing or unsupported versions produce typed `WorkflowFormatError` values
- raw identifiers, edge capacities, duplicate ports, missing nodes/ports, and cycles route through the existing domain validation path
- `WorkflowJsonError` separates JSON decode/encode failures from raw workflow format failures
- canonical JSON helpers are available:
  - `raw_workflow_from_json_str`
  - `workflow_from_json_str`
  - `raw_workflow_to_json_string`
- raw JSON rejects unknown fields and has round-trip tests
